# Auto-Update — Electron

Version: Electron 42.1.0, electron-updater 6.8.3 [S17, S18]

## Two Update Systems

| System | Package | Squirrel.Mac | Code signing req |
|--------|---------|-------------|-----------------|
| Built-in autoUpdater | `electron` (built-in) | YES (macOS) | YES |
| electron-updater | `electron-updater` (third-party) | NO | YES (macOS) |

Recommendation: Use **electron-updater** for new projects. More provider flexibility, better events, no Squirrel dependency, works with generic HTTP. [S18]

## electron-updater Setup

```bash
npm install electron-updater
```

```typescript
// src/updater.ts
import { autoUpdater } from 'electron-updater'
import log from 'electron-log'

export function setupUpdater(): void {
  autoUpdater.logger = log
  ;(log as any).transports.file.level = 'info'

  // Feed URL is set from app-update.yml (written at build time by electron-builder/forge)
  // DO NOT call autoUpdater.setFeedURL() — it reads from the bundled YAML

  autoUpdater.on('checking-for-update', () => {
    log.info('[updater] checking for update')
  })

  autoUpdater.on('update-available', (info) => {
    log.info('[updater] update available:', info.version)
    notifyUserUpdateAvailable(info)
  })

  autoUpdater.on('update-not-available', (info) => {
    log.info('[updater] no update available, current:', info.version)
  })

  autoUpdater.on('download-progress', (progress) => {
    log.info(
      `[updater] downloading: ${progress.percent.toFixed(1)}% ` +
      `(${(progress.bytesPerSecond / 1024).toFixed(0)} KB/s)`
    )
  })

  autoUpdater.on('update-downloaded', (info) => {
    log.info('[updater] update downloaded:', info.version)
    promptUserToRestart(info)
  })

  autoUpdater.on('error', (err) => {
    log.error('[updater] error:', err)
  })

  // Check at startup and periodically
  autoUpdater.checkForUpdatesAndNotify()
}

export function quitAndInstall(): void {
  autoUpdater.quitAndInstall()
}
```

## Provider: Generic (for Local Fixture Testing)

```typescript
// For testing — generic HTTP provider
const { NsisUpdater, MacUpdater } = require('electron-updater')

const provider = {
  provider: 'generic',
  url: 'http://localhost:8080/updates/', // local update server
}

// For programmatic config (testing without app-update.yml):
const updater = new MacUpdater(provider)
updater.forceDevUpdateConfig = true
```

For development testing without packaging, place a `dev-app-update.yml` at project root:

```yaml
# dev-app-update.yml
provider: generic
url: http://localhost:8080/updates/
updaterCacheDirName: my-app-updater
```

Enable with: `autoUpdater.forceDevUpdateConfig = true` [S18]

## Manifest Format (latest-mac.yml)

Auto-generated by electron-builder/electron-updater at publish time:

```yaml
version: 2.0.0
files:
  - url: MyApp-2.0.0.dmg
    sha512: <base64-sha512-of-dmg>
    size: 123456789
  - url: MyApp-2.0.0-mac.zip
    sha512: <base64-sha512-of-zip>
    size: 87654321
    blockMapSize: 12345
path: MyApp-2.0.0-mac.zip
sha512: <same-as-zip-sha512>
releaseDate: '2026-05-16T12:00:00.000Z'
releaseName: v2.0.0
releaseNotes: "Bug fixes and improvements"
stagingPercentage: 100
```

Fields:
- `version` — semver; must be higher than current to trigger update
- `files` — array of artifacts (ZIP is used for actual update; DMG for initial install)
- `path` / `sha512` — deprecated but present for compatibility with `>=2.15`
- `stagingPercentage` — 0-100; control rollout percentage [S18]

## Local Test Update Server

For testing, serve the manifest and artifacts from a local HTTP server:

```typescript
// scripts/local-update-server.ts
import { createServer } from 'node:http'
import { createReadStream, statSync } from 'node:fs'
import path from 'node:path'

const ARTIFACTS_DIR = path.join(__dirname, '../out/make/')
const PORT = 8080

createServer((req, res) => {
  const filePath = path.join(ARTIFACTS_DIR, req.url ?? '/')
  try {
    const stat = statSync(filePath)
    res.setHeader('Content-Length', stat.size)
    res.setHeader('Content-Type', 'application/octet-stream')
    createReadStream(filePath).pipe(res)
  } catch {
    res.statusCode = 404
    res.end('Not found')
  }
}).listen(PORT, () => {
  console.log(`Update server running at http://localhost:${PORT}`)
})
```

## quit-and-install Semantics

```typescript
autoUpdater.on('update-downloaded', () => {
  // Show dialog to user
  dialog.showMessageBox({
    type: 'info',
    title: 'Update Ready',
    message: 'A new version is ready to install.',
    buttons: ['Restart Now', 'Later'],
  }).then(({ response }) => {
    if (response === 0) {
      autoUpdater.quitAndInstall()
      // This calls app.quit() internally and installs the update
    }
  })
})
```

`quitAndInstall()`:
- On macOS: extracts the ZIP update and relaunches
- On Windows: runs the NSIS installer
- Does NOT fire `will-quit` event normally — flush critical state BEFORE calling it [S18]

## Built-in autoUpdater (Squirrel.Mac) — Reference Only

For comparison, the built-in approach:

```typescript
import { autoUpdater } from 'electron'

// Squirrel.Mac format

autoUpdater.setFeedURL({ url: feedUrl })

autoUpdater.on('update-downloaded', (event, releaseNotes, releaseName) => {
  autoUpdater.quitAndInstall()
})
```

## macOS Code Signing Requirement

`electron-updater` on macOS requires:
- The app to be code-signed with a Developer ID certificate
- The ZIP artifact must be included alongside the DMG (Squirrel.Mac-compatible)
- The code signature must match between update versions

Without code signing, `autoUpdater.checkForUpdatesAndNotify()` throws or silently fails on macOS. [S18]
