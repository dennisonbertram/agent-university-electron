# Simulated Signing — what would happen with real credentials

This file is auto-generated by Pulse's `forge.config.ts` `packageAfterCopy`
hook whenever `npm run package` runs without Apple Developer credentials in
the environment. Its presence is the deterministic test signal: no
`APPLE_ID`/`APPLE_TEAM_ID`/`APPLE_APP_SPECIFIC_PASSWORD` ⇒ signing +
notarization are skipped ⇒ this file is written ⇒ the skip-log marker
`packaging:signing:skipped:no-credentials` is also produced.

Skip-log token (the test grep target): `packaging:signing:skipped:no-credentials`

## What `forge` did (this run)

1. Bundled `dist/` into `out/<platform>-<arch>/<App>.app/Contents/Resources/app.asar`.
2. Applied `AutoUnpackNativesPlugin` (keeps `better-sqlite3`'s `.node`
   binary outside the asar so the native module can be loaded at runtime).
3. Applied `FusesPlugin` (RunAsNode=false, EnableNodeOptionsEnvironmentVariable=false,
   EnableCookieEncryption=true, OnlyLoadAppFromAsar=true,
   EnableEmbeddedAsarIntegrityValidation=true,
   EnableNodeCliInspectArguments=false).
4. Merged `Info.plist.template` into the final `Info.plist`
   (CFBundleURLTypes for `pulse://`, LSUIElement=true, CFBundleShortVersionString,
   NSHumanReadableCopyright).
5. SKIPPED osxSign — no `APPLE_ID` in env.
6. SKIPPED osxNotarize — no `APPLE_ID` in env.

The resulting bundle is launchable by Playwright (BT-C-12) but is NOT
distributable: macOS Gatekeeper will refuse to open it on a clean machine
("App is damaged" / "cannot verify developer").

## What would happen with real credentials

With `APPLE_ID`, `APPLE_TEAM_ID`, and `APPLE_APP_SPECIFIC_PASSWORD` set:

1. `@electron/osx-sign` signs every executable, framework, helper, and
   embedded dylib in the bundle — including the unpacked
   `app.asar.unpacked/node_modules/better-sqlite3/build/.../better_sqlite3.node`.
2. Forge then re-applies fuses to the signed binary.
3. `@electron/notarize` zips the signed bundle and submits to Apple's
   notarization service.
4. The notarization ticket is stapled to the bundle.

## What R-L5-4 / R-C regressions enforce

- `process.env.APPLE_ID` must remain near `osxSign`/`osxNotarize` in this
  file (no unconditional signing).
- The auto-unpack-natives plugin must remain present (R-C-8).
