# Simulated Signing — what would happen with real credentials

This file is auto-generated by `forge.config.ts`'s `packageAfterCopy` hook
whenever `npm run package` runs without Apple Developer credentials in the
environment. Its presence is the deterministic test signal for BT-L5-5:
no `APPLE_ID`/`APPLE_TEAM_ID`/`APPLE_APP_SPECIFIC_PASSWORD` ⇒ signing
+ notarization are skipped ⇒ this file is written ⇒ the skip-log marker
`packaging:signing:skipped:no-credentials` is also produced.

Skip-log token (the test grep target): `packaging:signing:skipped:no-credentials`

## What `forge` did (this run)

1. Bundled `dist/` into `out/<platform>-<arch>/<App>.app/Contents/Resources/app.asar`.
2. Applied `FusesPlugin` (RunAsNode=false, EnableNodeOptionsEnvironmentVariable=false,
   EnableCookieEncryption=true, OnlyLoadAppFromAsar=true,
   EnableEmbeddedAsarIntegrityValidation=true,
   EnableNodeCliInspectArguments=false).
3. Merged `Info.plist.template` into the final `Info.plist`
   (CFBundleURLTypes for `electron-l5://`, CFBundleShortVersionString,
   NSHumanReadableCopyright).
4. SKIPPED osxSign — no `APPLE_ID` in env.
5. SKIPPED osxNotarize — no `APPLE_ID` in env.

The resulting bundle is launchable by Playwright (BT-L5-10) but is NOT
distributable: macOS Gatekeeper will refuse to open it on a clean machine
("App is damaged" / "cannot verify developer").

## What would happen with real credentials

With `APPLE_ID`, `APPLE_TEAM_ID`, and `APPLE_APP_SPECIFIC_PASSWORD` set:

1. `@electron/osx-sign` signs every executable, framework, helper, and
   embedded dylib in the bundle with the Developer ID Application
   certificate.

   Equivalent manual command (per file):
   ```
   codesign --force --options runtime \
     --entitlements entitlements.mac.plist \
     --sign "Developer ID Application: <name> (<teamId>)" \
     out/<platform>-<arch>/<App>.app/Contents/MacOS/<exe>
   ```

2. Forge then re-applies fuses to the signed binary (fuse-flipping changes
   bytes; the post-flip binary must be re-signed). This is automatic via
   `FusesPlugin`.

3. `@electron/notarize` zips the signed bundle and submits to Apple's
   notarization service. Equivalent manual command:
   ```
   xcrun notarytool submit out/<App>.zip \
     --apple-id "$APPLE_ID" \
     --team-id "$APPLE_TEAM_ID" \
     --password "$APPLE_APP_SPECIFIC_PASSWORD" \
     --wait
   ```
   Timing: 30s to 30 minutes (typically 1-3 minutes in 2026).

4. On success, the ticket is stapled to the bundle:
   ```
   xcrun stapler staple out/<platform>-<arch>/<App>.app
   xcrun stapler validate out/<platform>-<arch>/<App>.app
   ```

5. The DMG made AFTER stapling inherits the staple. Verification:
   ```
   codesign --verify --deep --strict out/<App>.app
   spctl --assess --type exec --verbose out/<App>.app
   ```

## What `R-L5-4` enforces

The skip path must REMAIN conditional. If a future commit changes
`forge.config.ts` to call `osxSign` unconditionally, the regression test
fails (no `process.env.APPLE_ID` reference near `osxSign`/`osxNotarize`).
This protects against a CI run silently succeeding by signing with the
wrong key.
