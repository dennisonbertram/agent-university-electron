# L5 POC Report — Packaging, Code Signing, Auto-Update

## Goal

Produce a real packaged macOS app from the L4 menu-bar shell, document the
signing/notarization flow (simulated; no Apple Developer account on this
machine), wire `electron-updater` against a local-fixture HTTP server, and
prove `crashReporter` is started before the ready promise resolves.

## What landed

### electron-forge introduced (per Decision 1)

L1-L4 used a hand-rolled tsc + esbuild build chain because forge would
have buried the secure-defaults / IPC validation lessons of those POCs.
L5 introduces forge AS the packaging layer, while KEEPING the L4 build
chain. The forge package step runs `npm run build` first (via the
`prepackage` npm hook) and then bundles the produced `dist/` into the
`.app`. This avoids rewriting preload bundling for Vite's plugin pipeline
— a rewrite that would have had no behavior payoff for L5's test
surface. Documented as Decision 11 in `04_logs/decision-log.md`.

### Files added / modified

- `forge.config.ts` — packager + makers + plugins.
- `entitlements.mac.plist` — hardened-runtime entries.
- `Info.plist.template` — `CFBundleURLTypes` + version + copyright.

### `@signed-only` (not in suite — would require Apple Dev cert)

- `codesign --verify --deep --strict <App>.app` — would return 0 on a
  signed build. Cannot run.
- `xcrun stapler validate <App>.app` — would confirm notarization staple.
  Cannot run.
- `spctl --assess --type exec <App>.app` — Gatekeeper assessment.
  Cannot run.

These are documented in `simulated-signing.md` with the exact commands a
developer with creds would invoke.

## Decisions recorded

- **Decision 11 (`04_logs/decision-log.md`)** — Build chain: keep
  tsc + esbuild from L4, wrap with forge via `prepackage` / `premake`
  hooks. Alternative considered: scaffold a fresh Vite + electron-forge
  template and port the L4 source in. Rejected because the rewrite cost
  exceeds the behavior payoff for the L5 test surface, and because the
  preload-under-sandbox bundling pattern (L2 expectation gap entry 1) is
  load-bearing and well-tested with esbuild.

## Expectation gaps surfaced this POC

Five new entries appended to
`04_logs/expectation-gap-ledger.md`:

1. **Entry 7** — `electron-updater` cache-busting query param breaks naive
   path matching. The updater appends `?noCache=<random-token>` to its
   feed-URL GET. Servers that match `req.url.endsWith('/latest-mac.yml')`
   silently 404. Fix: strip `?…` before matching.
2. **Entry 8** — `packagerConfig.protocols` OVERRIDES `extendInfo.CFBundleURLTypes`
   instead of merging. The packaged plist's `CFBundleURLTypes` array
   comes entirely from `protocols`; the template's URL types are dropped.
   Workaround: declare the URL scheme via `protocols` only and use
   `extendInfo` for other Info.plist keys.
3. **Entry 9** — `electron-updater` refuses to check for updates in
   unpackaged dev mode by default. `forceDevUpdateConfig = true` is the
   undocumented escape hatch the project uses internally (the public
   doc covers it; the typedef does not expose it cleanly — we cast).
4. **Entry 10** — `playwright test` wipes the `test-results/` directory
   at the start of each run. Capturing verbatim output requires either a
   different sink path or capturing externally with `tee`.
5. **Entry 11** — Forge 7.11.1's `packageAfterCopy` hook signature
   accepts `(forgeConfig, buildPath)` for arity but the buildPath
   argument is the COPIED staging dir, not the final `.app` bundle
   directory. Writing the simulated-signing.md to `POC_ROOT` (the source
   tree) rather than the buildPath is intentional — we want the file
   tracked in git, not buried inside `out/`.

## Invariants for the capstone

The capstone ("Pulse") inherits L5's packaging shell. Items that MUST
continue to hold:

1. **Crash reporter pre-ready** (R-L5-1): the capstone adds `safeStorage`
   and Touch-ID prompts; both can crash. `crashReporter.start()` must
   stay above the ready promise so renderer-side crashes are caught.
2. **Fuses hardening** (R-L5-2): the capstone ships actual user data
   (encrypted journal entries). Fuses MUST be on; `RunAsNode:false`
   prevents the bundle being abused as a node interpreter.
3. **Generic-provider updater** (R-L5-3): the capstone is local-first;
   it has no remote update story for the school POC. Generic provider
   pointed at a fixture is the right shape for the capstone too.
4. **Signing skip path** (R-L5-4): the school machine has no creds; the
   capstone reuses the same skip-path artifact (`simulated-signing.md`).
5. **electron-l5 → pulse:// scheme** (R-L5-5): the capstone renames the
   scheme to `pulse://`. The same template/extendInfo merge pattern
   applies.

Additional invariants the capstone will need but that L5 doesn't enforce:

- `safeStorage.isEncryptionAvailable()` must be checked before encrypting
  rows (returns false on Linux without keyring; macOS dev keychain
  passes).
- `better-sqlite3` ABI must match the Electron ABI; the rebuild step is
  forge's `rebuildConfig` (currently `{}` in L5 — capstone will populate
  it with the better-sqlite3 module).
- Universal binary: the capstone should exercise BT-L5-4 since
  `better-sqlite3` ships per-arch binaries and a universal build forces
  both arches to be present.

## Recommended next step

**Capstone "Pulse"** — combines L1 (logger), L2 (secure IPC + CSP), L3
(SQLite + safeStorage), L4 (tray + notifications + shortcuts + power +
deep links + autolaunch + theme + dock), and L5 (forge packaging +
electron-updater + crashReporter). The L5 packaging shell is the bottom
layer for everything the capstone adds.

Concrete tasks for the capstone derived from L5 findings:

1. Rename deep-link scheme `electron-l5` → `pulse://`.
2. Add the focus-mode + journal subsystems (per `poc-plan.md` capstone
   section).
3. Populate `rebuildConfig` with the better-sqlite3 native module.
4. EXERCISE BT-L5-4 (universal binary) — the capstone is the right place
   to pay the 5-minute build cost and confirm both arches work with the
   real native dependencies.
5. Add `safeStorage` encryption for journal rows. Test the

   for the dock-icon-during-test affordance).
